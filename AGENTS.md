## Project Overview

ToolAlts is an **automated SEO content site** that discovers, verifies, and publishes software tool alternatives. The entire pipeline runs via GitHub Actions with zero manual intervention.

- **Domain**: https://www.toolalts.dev (primary) / https://toolalts.dev (redirects to www)
- **Deployed on**: Vercel (Next.js app with SSG public pages plus admin/API routes)
- **Content generated by**: DeepSeek API (via GitHub Actions)
- **Data sources**: GitHub API, Hacker News Algolia API, official websites

## Key Data Files

| File | Purpose |
|------|---------|
| `data/tools.json` | Master tool catalog (slug, name, tagline, description, category, pricing, features, rating, githubStars, etc.) |
| `data/comparisons.json` | Tool-to-tool comparison pairs (`{slug, toolA, toolB, category}`) |
| `data/categories.json` | Category definitions (`{slug, name, description}`) |
| `src/content/comparisons/*.md` | LLM-generated comparison articles |
| `src/content/reports/*.md` | AI verification audit reports |
| `src/content/blog/*.md` | Blog posts |

## Automation Scripts (in `scripts/`)

| Script | Trigger | What it does |
|--------|---------|--------------|
| `discover-tools.ts` | Every 4h (GitHub Actions) | Scans GitHub Search API + HN Show HN for new tools. Writes `data/pending-tools.json` |
| `ai-verify.ts` | After discover | Fetches GitHub data + website meta. Calls DeepSeek API for cross-validation. Writes `src/content/reports/*.md` + `data/verified-results.json` |
| `auto-approve.ts` | After verify | Reads verified results (confidence ≥70 + APPROVED). Appends to `data/tools.json`. Auto-generates new comparison pairs |
| `generate-comparisons.ts` | Weekly | Reads `data/comparisons.json`. For missing markdown, calls DeepSeek to generate `/compare/{a}-vs-{b}/` article |
| `sync-tools.ts` | Daily | Updates GitHub stars, last commit date for all tools in `data/tools.json` |
| `scrape-tool.ts` | Manual (workflow_dispatch) | CLI to manually add a new tool with Playwright screenshot |

## LLM Configuration

The project uses **DeepSeek API** via environment variables:

```
LLM_PROVIDER=deepseek
LLM_MODEL=deepseek-chat
LLM_API_KEY=sk-xxx
```

Set these in GitHub Actions → Secrets and variables.

If running locally, create `.env.local`:
```bash
LLM_PROVIDER=deepseek
LLM_API_KEY=sk-your-key
LLM_MODEL=deepseek-chat
```

## Data Types

```typescript
// src/lib/types.ts
interface Tool {
  slug: string;           // kebab-case ID
  name: string;           // display name
  tagline: string;        // one-line pitch
  description: string;    // 2-3 sentences
  category: string;       // slug from categories.json

  openSource: boolean;
  githubStars: number | null;

  websiteUrl: string;

  toolB: string;          // slug
  category: string;
}
```

## Common Tasks

### Add a new tool manually
```bash
npx tsx scripts/scrape-tool.ts --name "MyTool" --website "https://mytool.com" --github "user/repo" --category "productivity"
```

### Run discovery + verification locally
```bash
export LLM_PROVIDER=deepseek
export LLM_API_KEY=sk-xxx
export LLM_MODEL=deepseek-chat

npx tsx scripts/discover-tools.ts
npx tsx scripts/ai-verify.ts
npx tsx scripts/auto-approve.ts
```

### Build for production
```bash
npx next build
# Output goes to /out/ (static HTML)
```

### Serve locally
```bash
cd out && python3 -m http.server 8080
```

## Important Notes for Agents

1. **Never assume Next.js 14/15 conventions** — this is Next.js 16 with breaking changes. Always check `node_modules/next/dist/docs/` for current API.
2. **Scripts are excluded from TypeScript compilation** — `tsconfig.json` excludes `scripts/` directory. Scripts run via `tsx` directly.
3. **JSON data is source of truth** — `data/tools.json` and `data/comparisons.json` drive all pages. Editing these changes the site without code changes.
4. **Markdown content is auto-generated** — files in `src/content/` are produced by DeepSeek API. Don't manually edit unless fixing errors.
5. **Deployment shape** — `next.config.ts` currently uses `trailingSlash: true` and does not set `output: "export"`. Public pages are SSG where possible, while admin/API routes exist.
6. **Sitemap uses data layer** — `src/app/sitemap.ts` imports from `src/lib/tools.ts` and `src/lib/reports.ts`. If new data types are added, update sitemap too.
7. **SEO guardrail** — run `npm run seo:audit` before committing changes that affect public URLs, generated content, or catalog data.
