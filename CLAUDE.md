# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project Overview

ToolAlts is an **automated SEO content site** that discovers, verifies, and publishes software tool alternatives. The AutoClaw pipeline runs via GitHub Actions with zero manual intervention. All tool data lives in JSON files committed to the repo; blog, comparison, and report content is auto-generated by DeepSeek API.

- **Domain**: `https://www.toolalts.dev` (primary) / `toolalts.dev` redirects to www
- **Repo**: `focusontec/toolalts` on GitHub
- **Deploy**: Vercel (auto-deploys on push to `main`), DNS via Cloudflare

## Commands

```bash
npm run dev          # Dev server (Next.js, port 3000)
npm run build        # Build (outputs to .next, served by Vercel)
npm run lint         # ESLint

# Automation scripts (run with tsx, not through npm scripts)
npx tsx scripts/discover-tools.ts       # Discover new tools from GitHub + HN
npx tsx scripts/ai-verify.ts            # LLM-verify pending candidates
npx tsx scripts/auto-approve.ts         # Move approved tools into tools.json
npx tsx scripts/ai-audit.ts --mode all  # Run data accuracy + content quality audit
npx tsx scripts/ai-audit.ts --mode data --limit 5  # Audit specific subset
npx tsx scripts/generate-comparisons.ts # Generate comparison articles via LLM
npx tsx scripts/sync-tools.ts           # Batch-update GitHub stars
npx tsx scripts/scrape-pricing.ts --slug notion --dry-run  # Scrape real pricing from websites
npx tsx scripts/scrape-pricing.ts --slug linear            # Scrape and update pricing
npx tsx scripts/scrape-tool.ts --name "Tool" --website "https://..." --github "owner/repo"
```

## Architecture

### Next.js 16 with mixed rendering

Next.js 16 with `trailingSlash: true`. NOT a static export — the admin panel requires API routes and server components. Public pages are still statically generated via `generateStaticParams()`.

- `data/tools.json` — tool entries (flat array, NOT nested under a key)
- `data/comparisons.json` — comparison pairs (flat array of `{ slug, toolA, toolB, category }`)
- `data/categories.json` — category definitions
- `data/pending-tools.json` — candidates awaiting verification
- `data/verified-results.json` — LLM verification output (consumed by auto-approve, then cleared)
- `data/audit-results.json` — AI audit findings (data accuracy + content quality)
- `data/audit-fix-prompt.md` — auto-generated fix instructions for Claude Code
- `src/content/blog/` — blog Markdown with gray-matter frontmatter
- `src/content/comparisons/` — detailed comparison Markdown (auto-generated by DeepSeek)
- `src/content/reports/` — AI verification reports (generated by ai-verify)

### Page routes

| Route | Type | Source |
|-------|------|--------|
| `/` | Static | tools.json + categories.json |
| `/tool/[slug]/` | SSG | tools.json — detail page with pricing, features |
| `/compare/[slug]/` | SSG | comparisons.json + content/comparisons/*.md |
| `/category/[slug]/` | SSG | tools.json + categories.json |
| `/alternative-to/[slug]/` | SSG | tools.json |
| `/blog/[slug]/` | SSG | content/blog/*.md |
| `/reports/[slug]/` | SSG | content/reports/*.md |
| `/{ADMIN_PATH}/` | Dynamic | Admin panel (secret URL via ADMIN_PATH env var) |

### Admin panel

Secret URL path via `ADMIN_PATH` env var. Middleware rewrites the secret path to `/admin/` internally. Direct `/admin/` access is blocked.

- `src/middleware.ts` — path rewrite + auth cookie check
- `src/app/admin/layout.tsx` — server component, checks session cookie
- `src/app/admin/sidebar.tsx` — client component with navigation
- `src/app/api/admin/` — API routes for CRUD operations (use GitHub Contents API to write data)

### AutoClaw automation pipeline

5 GitHub Actions workflows:

1. **discover-tools.yml** (every 4h) — discover → verify → approve → generate → sync → commit
2. **sync-tools.yml** (twice daily) — update GitHub stars
3. **generate-comparisons.yml** (weekly) — generate comparison articles via LLM
4. **ai-audit.yml** (weekly) — data accuracy + content quality audit → commits audit-results.json + audit-fix-prompt.md
5. **scrape-new-tool.yml** (manual) — add new tool via CLI

LLM config: `LLM_PROVIDER=deepseek`, `LLM_API_KEY`, `LLM_MODEL=deepseek-chat`. Set in both GitHub Actions secrets AND Vercel env vars.

### Key lib modules

- `src/lib/tools.ts` — data access functions + blog post reader
- `src/lib/types.ts` — TypeScript interfaces: Tool, PricingPlan, Comparison, Category, BlogPost
- `src/lib/seo.ts` — JSON-LD generators
- `src/lib/affiliate.ts` — affiliate URL mapping
- `src/lib/reports.ts` — reads verification report Markdown
- `src/lib/admin.ts` — GitHub API helpers (getFileFromGitHub, updateFileOnGitHub, triggerWorkflow)

## Data schema (tools.json)

Each tool entry: `slug`, `name`, `tagline`, `description`, `rating`, `reviewsCount`, `openSource`, `githubStars`, `githubUrl`, `websiteUrl`, `pricing[]`, `features[]`, `category`, `logo`

`category` must match a slug in `data/categories.json`.

## Design system

- **Display font**: Instrument Serif (editorial feel)
- **Body font**: DM Sans
- **Colors**: Deep charcoal hero (`#1a1a2e`), warm cream surfaces (`#fefdfb`), coral accent (`#e8553d`)
- **Defined in**: `src/app/globals.css` via CSS custom properties (`--color-ink`, `--color-surface`, `--color-accent`, etc.)

## Important constraints

- **Trailing slashes everywhere** — all internal links and routes must use trailing slashes
- **Static params required** — public pages use `export const dynamicParams = false`
- **JSON data is source of truth** — editing `data/*.json` changes the site
- **Markdown content is auto-generated** — files in `src/content/` are produced by DeepSeek. Don't manually edit unless fixing errors
- **Scripts excluded from tsconfig** — scripts run via `tsx` directly
- **LLM jsonMode** — `callLlm()` defaults to JSON output. Pass `{ jsonMode: false }` for Markdown output (comparisons, blogs)
- **Anti-fabrication** — LLM prompts must include rules against inventing data (see ai-verify.ts for example)
