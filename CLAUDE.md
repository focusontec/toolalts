## Workflow rules

- Trailing slashes everywhere â€” all links and routes MUST end with `/`
- Public pages MUST have `export const dynamicParams = false`
- Scripts run via `tsx` directly, excluded from tsconfig
- `callLlm()` defaults to JSON output. Pass `{ jsonMode: false }` for Markdown
- LLM prompts MUST include anti-fabrication rules (never invent data)
- `data/*.json` is source of truth â€” editing these files changes the site
- `src/content/` is auto-generated by DeepSeek â€” don't manually edit unless fixing errors
- `category` in tools.json MUST match a slug in `data/categories.json`

## Data verification rules

- **MUST use Web Search (Ollama)** to verify all factual claims before writing: pricing, feature counts, limits, plan details
- Search queries should target official sources (e.g. "Slack pricing 2026", "Discord file upload limit")
- Never trust LLM training data for pricing or product specs â€” always verify against current web sources
- If web search returns no results, note the data as unverified in the content
