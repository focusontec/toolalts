# CLAUDE.md

## Commands

```bash
npm run dev          # Dev server (port 3000)
npm run build        # Production build
npm run lint         # ESLint

# Automation scripts (run with tsx, not npm)
npx tsx scripts/discover-tools.ts       # Discover tools from GitHub + HN
npx tsx scripts/ai-verify.ts            # LLM-verify pending candidates
npx tsx scripts/auto-approve.ts         # Move approved tools into tools.json
npx tsx scripts/generate-comparisons.ts # Generate comparison articles via LLM
npx tsx scripts/sync-tools.ts           # Batch-update GitHub stars
npx tsx scripts/ai-audit.ts --mode all  # Data accuracy + content quality audit
npx tsx scripts/scrape-pricing.ts --slug <name> --dry-run  # Scrape pricing
npx tsx scripts/ai-review.ts --dry-run  # Quality guardian review
```

## Workflow rules

- Trailing slashes everywhere — all links and routes MUST end with `/`
- Public pages MUST have `export const dynamicParams = false`
- Scripts run via `tsx` directly, excluded from tsconfig
- `callLlm()` defaults to JSON output. Pass `{ jsonMode: false }` for Markdown
- LLM prompts MUST include anti-fabrication rules (never invent data)
- `data/*.json` is source of truth — editing these files changes the site
- `src/content/` is auto-generated by DeepSeek — don't manually edit unless fixing errors
- `category` in tools.json MUST match a slug in `data/categories.json`

## Architecture notes

Next.js 16 with `trailingSlash: true`. NOT a static export — admin panel needs API routes + server components. Public pages are SSG.

Admin writes data via GitHub Contents API (`src/lib/admin.ts`). User submissions go to `data/user-submissions.json` (separate from auto-discovered `pending-tools.json`).

See @src/lib/types.ts for data schemas.
