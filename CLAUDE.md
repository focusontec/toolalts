## Project Overview

ToolAlts is an **automated SEO content site** that discovers, verifies, and publishes software tool alternatives. The AutoClaw pipeline runs via GitHub Actions with zero manual intervention. All tool data lives in JSON files committed to the repo; blog, comparison, and report content is auto-generated by DeepSeek API.

- **Domain**: `https://www.toolalts.dev` (primary) / `toolalts.dev` redirects to www
- **Repo**: `focusontec/toolalts` on GitHub
- **Deploy**: Vercel (auto-deploys on push to `main`), DNS via Cloudflare

## Commands

```bash
npm run dev          # Dev server (Next.js, port 3000)
npm run build        # Static export to out/
npm run lint         # ESLint

# Automation scripts (run with tsx, not through npm scripts)
npx tsx scripts/discover-tools.ts       # Discover new tools from GitHub + HN
npx tsx scripts/ai-verify.ts            # LLM-verify pending candidates
npx tsx scripts/auto-approve.ts         # Move approved tools into tools.json
npx tsx scripts/scrape-tool.ts --name "Tool" --website "https://..." --github "owner/repo"
npx tsx scripts/sync-tools.ts           # Batch-update GitHub stars + pricing screenshots
npx tsx scripts/generate-comparisons.ts # Generate comparison Markdown templates
npx tsx scripts/screenshot-tool.ts --slug <slug> --url <url>
npx tsx scripts/screenshot-pages.ts     # Screenshot local dev pages (hardcoded localhost:3002)
```

## Architecture

### Static site with JSON-driven data

Next.js 16 configured with `output: "export"` + `trailingSlash: true` — no server runtime, no SSR, no API routes. All pages are statically generated at build time via `generateStaticParams()`. This is Next.js 16, not 14/15 — there are breaking API changes (e.g. `params` is now a `Promise`).

- `data/tools.json` — tool entries (flat array, NOT nested under a key)
- `data/comparisons.json` — comparison pairs (flat array of `{ slug, toolA, toolB, category }`)
- `data/categories.json` — category definitions
- `data/pending-tools.json` — candidates awaiting verification (generated by discover-tools)
- `data/verified-results.json` — LLM verification output (consumed by auto-approve, then cleared)
- `src/content/blog/` — blog Markdown with gray-matter frontmatter
- `src/content/comparisons/` — detailed comparison Markdown (auto-generated by DeepSeek)
- `src/content/reports/` — AI verification reports (generated by ai-verify)
- `src/data/tool-manifest.json` — manually maintained build metadata

### Page routes (all static, all use trailing slashes)

| Route | Source |
|-------|--------|
| `/` | tools.json + categories.json |
| `/tool/[slug]/` | tools.json — detail page with pricing sidebar, features, affiliate link |
| `/compare/[slug]/` | comparisons.json + content/comparisons/*.md — side-by-side table + Markdown |
| `/category/[slug]/` | tools.json + categories.json |
| `/alternative-to/[slug]/` | tools.json — alternatives via comparisons.json lookup |
| `/blog/[slug]/` | content/blog/*.md |
| `/reports/[slug]/` | content/reports/*.md |
| `/sitemap.ts` | all slugs from tools, categories, comparisons, blog, reports |

### Key lib modules

- `src/lib/tools.ts` — all data access functions. Also reads blog posts from filesystem via gray-matter.
- `src/lib/types.ts` — TypeScript interfaces: Tool, PricingPlan, Comparison, Category, BlogPost
- `src/lib/seo.ts` — JSON-LD generators (SoftwareApplication, BreadcrumbList, ComparisonTable)
- `src/lib/affiliate.ts` — affiliate URL mapping (hardcoded per-tool slug)
- `src/lib/reports.ts` — reads AI verification report Markdown from content/reports/

### AutoClaw automation pipeline

Orchestrated by `.github/workflows/discover-tools.yml` (runs every 4 hours):

1. **discover-tools.ts** → GitHub Search API + HN Algolia → `data/pending-tools.json`
2. **ai-verify.ts** → fetches GitHub data + website meta, calls DeepSeek for cross-validation → `src/content/reports/*.md` + `data/verified-results.json`
3. **auto-approve.ts** → moves APPROVED entries (confidence ≥70) into tools.json, auto-generates comparison pairs → clears verified-results.json
4. Commit → Push → Vercel auto-deploy

LLM config via env vars: `LLM_PROVIDER=deepseek`, `LLM_API_KEY`, `LLM_MODEL=deepseek-chat`. Set in GitHub Actions secrets. For local dev, use `.env.local`.

## Data schema (tools.json)

Each tool entry must have these fields (see `src/lib/types.ts`):
`slug`, `name`, `tagline`, `description`, `rating`, `reviewsCount`, `openSource`, `githubStars`, `githubUrl`, `websiteUrl`, `pricing[]`, `features[]`, `category`, `logo`

`category` must match a slug in `data/categories.json`.

## Important constraints

- **Trailing slashes everywhere** — all internal links and routes must use trailing slashes (e.g. `/tool/notion/`)
- **Static params required** — pages use `export const dynamicParams = false`; all slugs must be pre-generated via `generateStaticParams()`
- **JSON data is source of truth** — editing `data/tools.json` or `data/comparisons.json` changes the site without code changes
- **Markdown content is auto-generated** — files in `src/content/` are produced by DeepSeek API. Don't manually edit unless fixing errors
- **Scripts excluded from tsconfig** — `tsconfig.json` excludes `scripts/` directory. Scripts run via `tsx` directly, not compiled by Next.js
- **Sitemap uses data layer** — `src/app/sitemap.ts` imports from `src/lib/tools.ts` and `src/lib/reports.ts`. Update sitemap if new data types are added
- **Tool logos are placeholders** — logo paths in tools.json are not rendered; the UI shows first-2-letter initials instead
- **Known issue**: `scrape-tool.ts` and `sync-tools.ts` have schema mismatches with current tools.json format and need fixing before use
